# AWS

Azure Web Services

**Cloud Coumputing** - доставка ІТ ресурсів за потребою за gay-as-you-go принципом.

**Multitenancy** - розподіл спільного заліза між віртуальними машинами.

**Vertial Scaling** - збільення потужності системи шляхом збільшення ресурсів інстансу.

**Horizontal Scaling** - Збільшення потужності системи шляхом збільшененя кількості серверів.

## Services

- **Computing**

  - **EC2** - (Amazon Elastic Compute Cloud) Віртуальні машини.

  - **EKS** - Managed Kubernetes Cluster, Кластер Кубернетес, яким керує AWS.

  - **ELB** - (Elastic Load Balancing) балансувальник навантаження який керуєится AWS.

  - **ECS** - Container Service. Сервіс який дозволяє деплойнути лише один контейнер.

  - **Lambda** - Дозволяє створювати Serverless застосунки.Оплата стягується за кількість запусків застосунку.

- **Storage**

  - **S3** - Дешевий сервіс для збереження даних.
  - **RDS** - PaaS сервіс для розгортання баз даних.

### EC2

Amazon EC2 (Elastic Compute Cloud) — це вебсервіс, який надає безпечні віртуальні сервери (інстанси) у хмарі з можливістю масштабування.

EC2 потребує `AMI` (Amazon Machine Image) для роботи. АМI можна отримати в 3 способи: завантажти власну, вибрати готову із магазину, купити платну із магазину.

EC2 ділиться на `types`, кожен із яких надає різні конфігурації інстансу (RAM, CPU, GPU, Storage, Networking)

Типи:

- **General Purpose** - Загальні для більшості завдань.
- **Compute Optimised** - Для складних обчислень.
- **Memory Optimized** - Для залежних від пам'яті задач.
- **Accelerated Computing** - Для обчислень, які вимагають участь графічного процесора.
- **Storage Optimized** - Для потреб у великому обсязі хранилища.

Billing:

- **Reserved Instances** - це підписка на хмарні ресурси, яка дозволяє отримати значні знижки в обмін на зобов’язання використовувати певний обсяг ресурсів протягом одного до трьох років.

- **On Denand Instances** - Оплата за умовою скільки використав стільки і заплатив.

- **Saving Instances** - договір про використання інстансу сталий час на місяці протягом n місяців за сталою ціною.

- **Spot Instances** - це невикористані хмарні ресурси, які продаються за значно зниженими цінами, але можуть бути відключені провайдером, коли ці ресурси займаються більш приорітетними ресурками, наприклад Reserved Instances, On Demand

- **Dedicated Hosts** - Оренда цілого фізичного сервера.

- **Dedicated Instances** - Послуга, яка гарантує, що сервер, на якому запущений шан ністанс не міститиме ресурси інших клієнтів.

Scaling:

- **Scalability** - здатність EC2 збільшувати як ресурси на одиницю, так і кількість одиниць
- **Elasticity** - здатність інстансу автоматично скалювати.

### ELB (Elastic Load Balancer)

Elastic Load Balancer - Це Ресурс який автоматично пиренаправлює трафік на інші ресурси і повністю керується AWS. Проводить постійно HealthChecks для перевірки життя інстансів.

Методи роутингу:

- **Round Robin** - Циклічно по всіх нодах.
- **Least Connections** - На той хто має найменше з'єднань.
- **IP Hash** - Перенаправляє тих самих користувачів на ту саму ноду.
- **Least Responce time** - Кидає на найшвидшу ноду.

### Messages

Для того щоб компоненти не були тісно пов'язані один із одним в, потрібно сервіс брокер, який прийматиме повідомлення від надавців до отримувачів.

- **SQS** - (Amazon Simple Queue Service) - Один компонент кладе повідомлення в чергу, а інший (воркер) періодично приходить і сам його забирає.

- **SNS** -  (Amazon Simple Notifitaion Service) - Повідомлення надсилається в тему (Topic), і SNS миттєво розсилає (штовхає) його копії всім підписникам одночасно.

- **EventBridge** - Він вміє заглядати всередину JSON-повідомлення. Завдяки гнучким правилам (Rules) він аналізує вміст і вирішує, куди саме направити подію.

### AWS Lambda

Lamba - serverless servise. Працює за принципом тригеррів. Спочатку виконується умова для тригеру, а Lambda запускає програму. Тобто Lambda є Event Driven. Максимальний час роботи AWS Lambda 15 хвилин. Оплата відбувається за час виконання і кількість запуску функції.

### ECS

Amazon Elastic Container Service - PaaS сервіс, за допомогою якого можна запустити лише контейнер. Може за основу мати EC2 або Fargate.

### EKS

Amazon Elastik Kubernetes - Сервіс який надає Kubernetes платформу для розгортання та керування великою кількістю контейнерів. AWS бере на себе керування Control Plane.

Для EKS є три опції розгортання нод:

- Self-Managed Nodes - вручну створені ноди користувачем
- Managed nodes - автоматично створені і керовані ноди AWS. Він автоматично створює Auto Scaling Group і масштабує ноди відноснот навантаження.
- Fargate - повний Serverless, не має поняття ноди, дорого

### ECR

Amazon Elastic Container Registry - Сервіс за допомогою якого ми можемо створити власне приватне сховище для контейнерів.

### Fargate

Amazon Fargate - Serverless сервіс, який повністю керується AWS, слугує для запуску контейнерів, може працювати як основа для ECS, EKS

### Elastic BeanStalk

Amazon Elastic BeanStalk - Serverless сервіс який спрощує деплой та управління аплікацією на EC2. Сервіс автоматично налаштовує VPC, EC2, ELB, роль користувача завантажити код та скофігурувати середовище

### Batch

Amazon Batch - Serverless сервіс для складних задач. Менеджить інфраструктуру та вміє автоматично скейлитись.

### Lightsail

Amazon Lightsail - serverless сервіс створений для веб застосунків. Коштує дешево, менеджить інфраструктру.

### Outpost

Amazon Outpost - сервер для створення гібридної інфраструктури, тобто поєднання особистої інфраструктури із AWS.

### CloudFront

AWS Cloudfront - сервіс AWS який кешує статичний контент для прискорення видалі клієнтам контенту. Для AWS використовує Edge Locations, тобто маленькі регіони із маленькими датацентрами які знаходяться поза звичайних регіонів і слугують для швидкої видачі контенту.

### CloudFormation

Amazon CloudFormation - сервіс для декларативного опису бажаної інфраструктури кодом.

## Networking

Нетворк для AWS.

### VPC

- **VPC** (Virtual Private Cloud) - Сервіс для створення віртуальних мереж в AWS в яких ми запускаємо наші ресурси.

- **Subnet** - підмережа VPC потрібна для організації різних ресурсів у різних частинах VPC. Загалом потрібна щоб в рамкхах лжнієї мережі робити частину ресурсів публічною а іншу приватною

- **Internet Gateway** - шлюз в мережу який надає можливість інстансам виходити в інтернет із своїх ІР та користувачам ззовні підключатись до цього інстансу. Щоб налаштувати вихід в інтернет потрібно створити запис в табличці маршрутизації із 0.0.0.0/0 на internet gateway

- **NAT Gateway** - потребує публічної ІР адреси. слугує для приватного вихіду в інтернет ресурсам в сабнеті, до якого NAT Gateway прикріплений, інстанси виходять в інтернет із ІР гейтвея.

- **Virtual Private Gateway** - приватний Gateway, слугує для обмеженого доступу вибраної кількості людей до приватної мережі. VPN сервер яким керує

#### VPN

- Client VPN - підключення одного користувача до VPC
- Site-to-Site VPN - підключення власної інфраструктури до VPC
- PrivateLink - це технологія, яка дозволяє безпечно підключати твій VPC до сервісів AWS, сторонніх SaaS-платформ (наприклад, Snowflake, Datadog) або твоїх власних додатків в інших VPC, взагалі не випускаючи трафік в інтернет.
- Direct Connect - пряме підключення кабелем до дата центру AWS

#### Firewall

- Security Groups - це фаєрвол для інстансів (віртуальних машин/мережевих карток ENI). Він statefull, тобто запам'ятовує який пакет зайшов в інстанс і автоматично його випускає.
- Network ACL - це фаєрвол для сабнетів (підмереж). Він stateless, тобто не запам'ятовує який трафік зайшов і який вийшов, тому потрібно чітко прописувати Inbound і Outbound rules.

Загалом краще мати ACL відкритим а в Security Groups уже прописувати усі чіткі правила.
